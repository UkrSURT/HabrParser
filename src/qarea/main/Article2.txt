    <h1 class="post__title">
      <span class="post__title-text">Разработка для Sailfish OS: Тестирование QML-кода, зависящего от C++ в Sailfish OS</span>
      







<span class="flag flag_tutorial"  title="Обучающий материал">tutorial</span>



    </h1>

      <div class="hubs">
          <a href="https://habrahabr.ru/hub/mobile_testing/" class="hub " title="Вы не подписаны на этот хаб" onclick="if (typeof ga === 'function') { ga('send', 'event', 'hub', 'post page', 'Тестирование мобильных приложений'); }">Тестирование мобильных приложений</a><span class="profiled_hub" title="Профильный хаб">*</span>, 
          <a href="https://habrahabr.ru/hub/mobile_dev/" class="hub " title="Вы не подписаны на этот хаб" onclick="if (typeof ga === 'function') { ga('send', 'event', 'hub', 'post page', 'Разработка мобильных приложений'); }">Разработка мобильных приложений</a><span class="profiled_hub" title="Профильный хаб">*</span>, 
          <a href="https://habrahabr.ru/hub/qt_software/" class="hub " title="Вы не подписаны на этот хаб" onclick="if (typeof ga === 'function') { ga('send', 'event', 'hub', 'post page', 'Qt'); }">Qt</a><span class="profiled_hub" title="Профильный хаб">*</span>
      </div>

  <div class="post__body post__body_full">
    <div class="content html_format js-mediator-article">Здравствуйте! Данная статья является продолжением цикла статей, посвященных разработке для мобильной платформы Sailfish OS и их тестированию. <a href="https://habrahabr.ru/post/320808/">Одна из предыдущих статей</a> была посвящена тестированию QML-компонентов приложения. Однако, часто разработчики сталкиваются с потребностью написания собственных компонентов на языке C++ для использования функционала, недоступного из QML, или для улучшения производительности. Об этом также <a href="https://habrahabr.ru/post/321426/">уже было написано</a>. Тестирование таких компонентов отличается от тестирования уже существующих. В данной статье мы расскажем, как тестировать собственные QML-компоненты, написанные на языке C++.<br>
<a name="habracut"></a><br>
<h3>Тестируемое приложение</h3><br>
В качестве приложение возьмём то же самое, что брали в <a href="https://habrahabr.ru/post/320808/">предыдущей статье</a> — приложение-счётчик (исходный код приложения доступен на <a href="https://github.com/ExAnimoso/counter-application">GitHub</a>). В текущей реализации значение счётчика хранится в свойстве страницы. Это значение изменяется при нажатии на кнопки «add» и «reset» и используется для отображения на виде.<br>
<br>
Мы изменим приложение таким образом, чтобы значение хранилось и изменялось с помощью кода, написанного на C++. Добавим класс <i>Counter</i>, имеющий следующий вид:<br>
<br>
<pre><code class="cpp">// counter.h
#include &lt;QObject&gt;

class Counter : public QObject {
    Q_OBJECT
    Q_PROPERTY(int count READ count NOTIFY countChanged)
private:
    int m_count = 0;
public:
    int count();
    Q_INVOKABLE void incrementCount();
    Q_INVOKABLE void resetCount();
signals:
    void countChanged();
};
</code></pre><br>
<br>
<pre><code class="cpp">// counter.cpp
int Counter::count() {
    return m_count;
}

void Counter::incrementCount() {
    m_count++;
    emit countChanged();
}

void Counter::resetCount() {
    m_count = 0;
    emit countChanged();
}
</code></pre><br>
Данный класс унаследован от <i>QObject</i> и содержит в своём теле макрос <i>Q_OBJECT</i>, что позволяет экспортировать его в QML. Класс содержит единственное свойство <i>count</i>, доступное только для чтения. Для изменения значения этого свойства используются методы <i>incrementCount()</i> и <i>resetCount()</i>. Методы помечены макросом <i>Q_INVOKABLE</i>, для того, чтобы их можно было вызывать из QML.<br>
<br>
Для использования классов, написанных на C++ в QML, необходимо зарегистрировать класс как QML-тип. Обычно это делается в теле функции <i>main()</i> приложения. В данном случае мы этого сделать не можем. Дело в том, что <i>qmltestrunner</i> не вызывает главную функцию приложения при запуске тестов.<br>
<br>
<h3>QML-плагин</h3><br>
Решить проблему, описанную выше, можно с помощью <a href="http://doc.qt.io/qt-5/qtqml-modules-cppplugins.html">QML-плагина</a>. Идея состоит в том, чтобы выделить C++ код в библиотеку и использовать её при запуске приложения и тестов. Регистрация QML-типов в данном случае переносится в QML-плагин.<br>
<br>
Для того, чтобы создать плагин, необходимо разделить проект на две части. Под каждую из частей создадим директории с подпроектами, содержащими файл <i>*.pro</i> с названием, соответствующим названию директории. Первый подпроект назовём <b>core</b>, он будет содержать весь C++ код, кроме файла <i>*.cpp</i> с функцией <i>main()</i>. Второй — <b>app</b>, он будет содержать QML-файлы, файл <i>*.desktop</i> и файлы с переводами. Сюда же поместим <i>*.cpp</i> файл с функцией <i>main()</i>.<br>
<br>
Файл <i>core.pro</i> выглядит следующим образом:<br>
<br>
<pre><code>TEMPLATE = lib

TARGET = core

CONFIG += qt plugin c++11

QT += qml\
    quick\

HEADERS += \
    counter.h

SOURCES += \
    counter.cpp

DISTFILES += qmldir

uri = counter.cpp.application.Core

qmldir.files = qmldir
installPath = /usr/lib/counter-cpp-application/$$replace(uri, \\., /)
qmldir.path = $$installPath
target.path = $$installPath
INSTALLS += target qmldir
</code></pre><br>
В файле <i>core.pro</i> необходимо использовать <i>TEMPLATE = lib</i>, чтобы подпроект использовался как библиотека. Целью библиотеки (<i>TARGET</i>) указываем её название, т.е. <i>core</i>. В <i>CONFIG</i> обязательно добавляем <i>plugin</i>, чтобы указать, что библиотека подключается в виде плагина. В <i>DISTFILES</i> необходимо добавить путь к файлу <i>qmldir</i>, который должен быть расположен в директории <i>core</i> и содержать название модуля и плагина. В нашем случае будет следующее содержимое этого файла:<br>
<br>
<pre><code>module counter.cpp.application.Core
plugin core
</code></pre><br>
В конце файла <i>core.pro</i> необходимо указать путь к файлам библиотеки и к файлу <i>qmldir</i>. Библиотеку помещаем в директорию <i>/usr/lib/</i>, а путь к плагину указываем <i>counter/cpp/application/Core</i>.<br>
<br>
Файл <i>app.pro</i> содержит конфигурацию самого приложения. Здесь указывается целевое приложение, пути к файлам qml, иконок и переводов. Сюда же добавляем <i>*.cpp</i> файл с главной функцией. В нашем примере файл имеет следующий вид:<br>
<br>
<pre><code>TARGET = counter-cpp-application

CONFIG += sailfishapp \
          sailfishapp_i18n \
          c++11

SOURCES += src/counter-cpp-application.cpp

OTHER_FILES += qml/counter-cpp-application.qml \
    qml/cover/CoverPage.qml \
    translations/*.ts \
    counter-cpp-application.desktop

TRANSLATIONS += translations/counter-cpp-application-de.ts

SAILFISHAPP_ICONS = 86x86 108x108 128x128 256x256

DISTFILES += \
    qml/CounterCppApplication.qml \
    qml/pages/CounterPage.qml 
</code></pre><br>
Теперь необходимо изменить <i>*.pro</i> файл проекта, чтобы он включал в себя два созданных нами подпроекта. Для этого добавим в данный файл <i>TEMPLATE = subdirs</i> и добавим наши директории как подпроекты. В таком случае по очереди будут собраны подпроекты, находящиеся в данных директориях. Здесь же необходимо оставить добавление файлов проекта из директории <i>rpm</i>, так как они всегда должны находиться в корне основного проекта. Для нашего приложения это выглядит так:<br>
<br>
<pre><code>TEMPLATE = subdirs

OTHER_FILES += $$files(rpm/*)

SUBDIRS += \
    app \
    core

app.depends = core
</code></pre><br>
Здесь же мы указали, что подпроект <b>app</b> зависит от <b>core</b>.<br>
<br>
Теперь, когда структура проекта подготовлена, можно приступать к реализации плагина. Необходимо создать C++ класс, который мы назовём <i>CorePlugin</i>. Класс должен быть унаследован от <a href="http://doc.qt.io/qt-5/qqmlextensionplugin.html">QQmlExtensionPlugin</a>. В нём будет производиться регистрация типов и инициализация движка.<br>
<br>
В классе QQmlExtensionPlugin есть два метода, которые можно переопределить:<br>
<br>
<ul>
<li><i>registerTypes(const char *uri)</i>: нужен для регистрации наших QML-типов. В качестве параметре методу передаётся URI нашего плагина, который указывается в файле <i>qmldirs</i>.</li>
<li><i>initializeEngine(QQmlEngine *engine, const char *uri)</i>: нужен для инициализации движка нашего приложения. Первый параметр является QML-движком, с помощью которого можно настроить приложение, второй — URI плагина.</li>
</ul><br>
В нашем случае пока достаточно только первого метода. Используем его для регистрации класса <i>Counter</i>. В тело метода <i>registerTypes()</i> поместим следующее:<br>
<br>
<pre><code>qmlRegisterType&lt;Counter&gt;(uri, 1, 0, "Counter");
</code></pre><br>
В угловых скобках указывается имя регистрируемого класса. Первым параметром передаётся URI плагина. Вторым и третьим параметрами передаются старший и младший номера версий. Четвёртым параметром передаётся имя, под которым будет доступен наш класс из QML. Теперь можно использовать тип Counter в нашем приложении.<br>
<br>
<h3>Использование собственного QML-компонента</h3><br>
Теперь нам необходимо указать путь к библиотеке в главной функции приложения. Для этого нам необходимо вручную инициализировать приложение и вид. В обычном случае инициализация производится следующим образом:<br>
<br>
<pre><code class="cpp">int main(int argc, char *argv[]) {
    return SailfishApp::main(argc, argv);
}
</code></pre><br>
Изменим код функции <i>main()</i> для того, чтобы указать путь к библиотеке:<br>
<br>
<pre><code class="cpp">int main(int argc, char *argv[]) {
    QGuiApplication* app =  SailfishApp::application(argc, argv);
    QQuickView* view = SailfishApp::createView();
    view-&gt;engine()-&gt;addImportPath("/usr/lib/counter-cpp-application/");
    view-&gt;setSource(SailfishApp::pathTo("qml/counter-cpp-application.qml"));
    view-&gt;showFullScreen();
    QObject::connect(view-&gt;engine(), &amp;QQmlEngine::quit, app, &amp;QGuiApplication::quit);
    return app-&gt;exec();
}
</code></pre><br>
Здесь мы создаём экземпляры приложения и вида. Затем указываем путь к нашей библиотеке, путь к главному QML-файлу приложения и отображаем вид на весь экран. В конце концов настраиваем обработчик закрытия приложения и запускаем его. Теперь можно импортировать созданную библиотеку в QML-файлах и использовать тип <i>Counter</i>.<br>
<br>
За основу возьмём файл <i><a href="https://github.com/ExAnimoso/counter-application/blob/master/qml/pages/CounterPage.qml">CounerPage.qml</a></i> из предыдущей статьи. Добавим наш плагин следующим образом:<br>
<br>
<pre><code class="javascript">import counter.cpp.application.Core 1.0
</code></pre><br>
Здесь используется URI, ранее указанный в файле <i>qmldirs</i>, и версии 1.0 указанные при регистрации типа. Для использования типа добавим его внутри страницы:<br>
<br>
<pre><code class="javascript">Counter {
    id: counter
}
</code></pre><br>
Теперь вместо изменения значения свойства count будем вызывать методы <i>counter.increment()</i> и <i>counter.reset()</i> при добавлении и сбросе счётчика соответственно.<br>
Код тестов при этом остаётся тем же самым, что и в предыдущей статье, так как мы не меняли визуальные компоненты.<br>
<br>
<h3>Запуск тестов</h3><br>
Запуск тестов происходит с помощью <i>qmltestrunner</i>. Так как часть кода перешла в QML-плагин, то нам необходимо вручную указывать путь к нему. Для этого используется переменная <i>QML2_IMPORT_PATH</i>, которой присваивается путь к файлам библиотеки перед запуском тестов. В итоге для рассматриваемого приложение это будет выглядеть так:<br>
<br>
<pre><code>QML2_IMPORT_PATH=/usr/lib/counter-cpp-application/ /usr/lib/qt5/bin/qmltestrunner -input /usr/share/counter-cpp-application/tests/
</code></pre><br>
Код тестов и вывод результатов остаётся тем же самым, что и в <a href="https://habrahabr.ru/post/320808/">предыдущей статье</a>:<br>
<br>
<pre><code>********* Start testing of qmltestrunner ********* 
Config: Using QtTest library 5.2.2, Qt 5.2.2 
PASS : qmltestrunner::Counter tests::initTestCase() 
PASS : qmltestrunner::Counter tests::test_counterAdd() 
PASS : qmltestrunner::Counter tests::test_counterReset() 
PASS : qmltestrunner::Counter tests::cleanupTestCase() 
Totals: 4 passed, 0 failed, 0 skipped 
********* Finished testing of qmltestrunner *********
</code></pre><br>
<br>
<h3>Заключение</h3><br>
Тестирование собственных QML-компонентов отличается от тестирования уже имеющихся в библиотеке QtQuick. Для реализации возможности их тестирования нам пришлось выделить весь C++ код в QML-плагин и подключать его как библиотеку. В результате код для тестирования не отличается от того, который тестировал бы стандартные QML-компоненты. Тем не менее сам проект потребовал значительных изменений. Приложение в таком виде уже можно использовать при публикации в магазине. Исходный код примера доступен на <a href="https://github.com/ExAnimoso/counter-cpp-application">GitHub</a>.<br>
<br>
Технические вопросы можно также обсудить на <a href="https://t.me/sailfishos">канале русскоязычного сообщества Sailfish OS в Telegram</a> или <a href="https://vk.com/merproject">группе ВКонтакте</a>.<br>
